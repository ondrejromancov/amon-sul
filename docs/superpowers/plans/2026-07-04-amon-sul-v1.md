# Amon Sûl v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A self-hostable, read-only GCP fleet dashboard (per `docs/superpowers/specs/2026-07-04-amon-sul-design.md`) that runs locally in mock mode with zero credentials, with live GCP collectors ready for a config + ADC.

**Architecture:** npm-workspaces monorepo. `packages/shared` holds domain types. `apps/server` (Fastify) owns config, a poller filling an in-memory `FleetStore`, REST + SSE, and swappable collectors (mock fixtures vs GCP APIs). `apps/web` (Vite + React) renders one screen ported pixel-for-pixel from `docs/prototype/gcp-fleet-prototype.html`.

**Tech Stack:** Node 24, TypeScript, npm workspaces, Fastify 5, zod, yaml, googleapis / @google-cloud clients, Vite 6, React 19, Vitest, @testing-library/react.

## Global Constraints

- Read-only v1: no write operations; the prototype's config-editing drawer section and "Save changes" flow are dropped; the "$41/mo" cost stat tile is dropped.
- Mock mode when `AMON_SUL_MOCK=1` OR config file missing; snapshot carries `mode: 'mock'`; header shows a "mock data" badge.
- Layout (node x/y px + board w/h) resolved server-side; web is a dumb renderer.
- Resource id format: `${projectId}/${type}/${name}`. Config shorthand keys: `type/name`.
- No state library, no Tailwind, no router; plain CSS ported from the prototype.
- Server default port 8787 (dev), single port serves API + built SPA in prod (8080 in Docker).
- All timestamps ISO 8601 strings.

**Interfaces:**
