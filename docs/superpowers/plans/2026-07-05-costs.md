# Cost Visibility Implementation Plan

**Goal:** Spot what each resource/project costs per month, with overview breakdown graphs.

**Reality check:** GCP exposes no "spend per project" REST endpoint. Accurate costs come only
from a BigQuery billing export, which the owner's account does not currently have. Therefore:

- **Tier 1 (default): estimates.** A pure, tested pricing table maps discovered resources to
  ~monthly USD (SQL tiers, GCE machine types, Redis GB, GCS GB via a per-project Monitoring
  query for bucket sizes, Cloud Run min-instances). Every figure is labeled _estimated_.
- **Tier 2 (optional): billing export.** `billing.bigqueryTable` in config â†’ server queries
  monthly cost per project/service for the last 6 months (BigQuery `jobs.query`), giving a
  real trend chart and actual project totals. Mapper unit-tested against fixture rows.

## Web

- Node cards: cost appended to the metrics row (`~$28/mo`).
- Project label: rolled-up `~$X/mo`.
- **Costs view**: a "Costs" toggle in the top bar swaps the canvas panel for an overview:
  - stat tiles: fleet total/mo (est or actual), top project, top category
  - stacked horizontal bars: per project, colored by resource category (existing palette)
  - bar list: per service type across the fleet
  - trend columns (only when billing months exist): last 6 months total
  - per-resource table sorted by cost desc, notes shown
  - persistent "estimated from list prices â€” enable billing export for actuals" footnote
